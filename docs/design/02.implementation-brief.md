## Project goal

Build a static skeleton website for the GUARDIANS public web presence based on `01.information-architecture.md` ("IA").

## Tech stack

- Use Astro
- GitHub pages deployment as a static site build

## Requirements

- Implement five routes: `/`, `/services-resources`, `/partners-contributions`, `/about`, `/contact`.
- Use the IA as the source of truth for page sections, navigation, CTAs, controlled vocabularies and cross-linking.
- Use [BioCommons-inspired](https://www.biocommons.org.au/) visual language.
- Use the content in `./content-background` for draft text and image content.
- Keep the homepage as an orientation page; do not feature selected catalogue items there.
- Implement catalogue cards using the schema in the IA.
- Implement project-level contribution entries using the schema in the IA, with lead partner organisation logos.
- Implement a site-wide footer with contact information and funder acknowledgement, including BPA and NCRIS attribution where approved assets are available.
- Implement simple client-side filtering using Astro with js islands.
- Add visible status and visibility/access labels to catalogue cards.
- Cross-link catalogue items and partner entries where sample data allows.
- Ensure the project builds successfully.

## Implementation principles

- Prefer small, reviewable changes.
- Before the first commit of each day, do a project-wide review checking for overall integrity. Identify technical debt and cruft e.g. dead code, structural or logical duplication, inconsistent handling of data and events across modules.
- Use semantic, accessible HTML.
- Use responsive layouts.
- Keep content data as markdown separate from presentation components, as per standard Astro patterns.
- Do not use lorem ipsum.
- Do not invent authoritative content. Clearly mark any draft content inferred from `./content-background/` as such. For draft content that cannot be inferred from the background materials, prefer empty data structures and render empty-state UI.
- Keep the visual design clean and restrained; do not over-brand.

## Logo asset handling

- Use logo image files cropped to the minimal visible bounding box where the source asset has a transparent background.
- Do not add visual padding inside logo files to make them appear balanced in a particular layout.
- Handle layout balance through CSS tile sizing, image max-width/max-height rules, and per-logo presentation classes where needed.
- Preserve official logo artwork and proportions; do not stretch, recolour, or crop into the visible mark.
- Logos with non-transparent backgrounds or fixed brand containers, such as the University of Melbourne mark, may remain as supplied and be balanced with CSS.
